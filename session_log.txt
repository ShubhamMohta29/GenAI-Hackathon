SESSION 0 — Hackathon (13–15 March 2026) — Initial build
--------------------------------------------------------------------------------
Branch: main / various feature branches (PRs #1–#15)

  INFRASTRUCTURE
  - Scaffolded project: backend/ (api, data, model, graph) + frontend/
  - Added .gitignore, raw_data gitignore, .env.example
  - Added Kaggle downloader script (generate_data.py) with token setup
  - Added setup.bat / setup.sh (Python venv + npm install)
  - Added run.bat / run.sh (launch both servers)
  - Added install_prereqs.bat (Python, Node, Neo4j Desktop installers — later removed)

  MODEL
  - Built PyG graph from PaySim CSV: node features, edge features, fraud labels
    (backend/model/dataset.py + backend/data/graph_builder.py — duplicate, later removed)
  - Implemented FraudGNN: 3-layer GraphSAGE with SAGEConv (backend/model/gnn.py)
  - Training script: 200 epochs, class-weighted NLL loss, 80/20 split
    (backend/model/train.py) — uses sample_size=100000 for speed
  - Evaluation metrics: Precision, Recall, F1 on test set
  - Exports fraud_gnn.pt (weights) + scores.json (per-node probabilities)

  BACKEND
  - Initial FastAPI setup with Neo4j integration (later ripped out)
  - Rewrote backend to remove Neo4j: all queries run against Pandas DF in memory
    (commit 06817709)
  - Endpoints: /alerts, /accounts/high, /accounts/medium, /accounts/low,
    /graph/{account_id}, /profile/{account_id}, /rings, /profile/ring/{ring_id}
  - Ring detection: NetworkX connected components on fraud edges
  - AI SAR generation: Google Gemini 2.5 Flash Lite, structured prompt
  - WebSocket /ws/live: streams real PaySim rows with GNN risk scores (2s interval)
  - Timestamps: PaySim step → readable date string via step_to_timestamp()
  - Live transactions: switched from random to real dataset rows (commit dd6f479e)

  FRONTEND
  - React 19 + Vite 8 dashboard, 3-panel layout
  - Force-directed graph: react-force-graph-2d, colored by risk tier
  - Custom canvas node renderer: glow, risk % badge, size by degree
  - Alert cards (top 15), Clusters tab, ProfileCard SAR viewer
  - Branding updated: FraudLink → ARGUS (commit 38b2ad98)
  - ngrok bypass headers added to all fetch calls

  DOCUMENTATION
  - README.md: setup guide, API reference, tech stack
  - HackathonWriteUp.md: project description and submission narrative
