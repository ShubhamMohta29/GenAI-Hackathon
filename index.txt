--------------------------------------------------------------------------------
WHAT THIS APP IS
--------------------------------------------------------------------------------
Argus is a real-time Anti-Money Laundering (AML) fraud investigation platform
built for the GenAI Genesis Hackathon (13-15 March 2026), sponsored prize: TD.

It combines a custom-trained Graph Neural Network (GraphSAGE) with Google
Gemini AI to let fraud analysts identify suspicious accounts, visualize
transaction networks, detect fraud rings, and generate Suspicious Activity
Reports (SARs) — all from one dashboard.

--------------------------------------------------------------------------------
API ENDPOINTS (backend/api/main.py)
--------------------------------------------------------------------------------
GET  /alerts                 Top 20 highest-risk accounts (risk > 0.7)
GET  /accounts/high          All accounts risk > 0.7 (capped at 200 in response)
GET  /accounts/medium        All accounts risk 0.4–0.7 (capped at 200)
GET  /accounts/low           Count only of low-risk accounts (too many to list)
GET  /graph/{account_id}     1-hop transaction neighborhood (edges capped at 200)
GET  /profile/{account_id}   AI-generated SAR via Gemini for a single account
GET  /rings                  Top 20 suspicious transaction clusters (rings)
GET  /profile/ring/{ring_id} AI-generated SAR via Gemini for a cluster
WS   /ws/live                Simulated live transaction stream (2s interval)

--------------------------------------------------------------------------------
KNOWN BUGS / ISSUES
--------------------------------------------------------------------------------
1. SECURITY: backend/.env contains a live GEMINI_API_KEY in plaintext.
   Although gitignored, the key should be rotated immediately if the repo
   was ever pushed with it included.

2. WRONG ENV PATH: backend/graph/neo4j_loader.py loads dotenv from
   "../data/.env" but .env is actually at backend/.env. Will crash if run.

3. DEAD CODE: backend/data/graph_builder.py duplicates model/dataset.py
   exactly. The model uses dataset.py; graph_builder.py is unreferenced.

4. VITE BOILERPLATE: frontend/src/App.css is the default Vite template CSS
   (hero, ticks, counter classes). None of these classes exist in App.jsx.
   The actual layout CSS is in index.css. App.css should be cleaned out.

5. SCORE COVERAGE GAP: scores.json only covers the sampled 100k-row
   subset. Accounts not in the sample get SCORES.get(account_id, 0.0) → 0.0
   (treated as no-risk). This silently misclassifies unknown accounts.

6. RING PROFILE ROUTING CONFLICT: FastAPI routes /profile/{account_id} and
   /profile/ring/{ring_id} — "ring" will be matched as account_id by the
   first route if requests come in the wrong order. FastAPI resolves this by
   route definition order (ring route is defined after account route), but
   it is fragile. The ring endpoint should be moved before the account one.

7. train.py MODIFIED (Shubham branch): backend/model/train.py has uncommitted
   changes on the current branch. Review and commit or discard.

8. VITE_API_URL in frontend/.env points to dead ngrok URL — must be changed
   to localhost:8000 for local development.
